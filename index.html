Problem

Understanding processor design requires going beyond theory — I needed to implement a complete instruction execution path from fetch to write-back to internalize how modern CPUs actually work.

Engineering Challenge

Pipeline hazards (data, control, structural) create correctness risks. Balancing pipeline depth against clock frequency while maintaining correct instruction ordering demanded careful hazard detection and forwarding architecture.

Design Process

Started with ISA specification and datapath partitioning. Mapped each instruction class to pipeline stages, then designed control signals, hazard unit, and forwarding paths before integration testing.

Engineering Decisions

Chose a classic five-stage pipeline over deeper superscalar design for verifiability. Implemented forwarding before stalling where possible to minimize pipeline bubbles. Prioritized waveform inspection over automated coverage metrics given project scope.

Future Development

FPGA synthesis on target board, branch prediction unit, cache hierarchy integration, and expanded testbench with automated regression.

Problem

Most people lack visibility into daily water usage until a bill arrives. Without real-time feedback, conservation behavior is hard to sustain.

Engineering Challenge

Designing a system that is accurate enough to change behavior, affordable to produce, and simple enough that non-technical users will actually adopt it.

Design Process

Identified target use cases, mapped sensor placement options, prototyped data display concepts, and evaluated trade-offs between installation complexity and measurement precision.

Technical Implementation

Explored flow-rate sensing approaches, microcontroller data logging, and a clear feedback interface. Focused on modularity so hardware and UX could evolve independently.

Engineering Decisions

Prioritized user experience over sensor sophistication — a readable daily summary matters more than raw data granularity for behavior change.

Future Development

Prototype with embedded MCU, mobile dashboard, calibration workflow, and field testing with real households.

Problem

A Honda Civic developed recurring overheating under load. A BMW E39 540i required systematic maintenance to remain reliable as a daily driver.

Engineering Challenge

Automotive failures rarely have a single cause. Overheating involved tracing the coolant loop, verifying thermostat operation, inspecting radiator flow, and ruling out head gasket issues methodically.

Design Process

Applied fault isolation: observe symptoms, form hypotheses, test each subsystem, document findings, and verify the fix under the original failure conditions.

Technical Implementation

  • Honda Civic — overheating investigation: coolant level audit, thermostat testing, radiator inspection, hose and clamp evaluation
  • BMW E39 540i — preventive maintenance: fluid services, cooling system checks, wear-item replacement scheduling

Engineering Decisions

Chose systematic elimination over parts-replacement guessing. Documented each test result before moving to the next subsystem — the same discipline applied in hardware debug.

Future Development

OBD-II data logging integration, thermal imaging for cooling diagnostics, and structured maintenance logs for predictive servicing.

Problem

Building a product people actually want requires more than a good idea — it demands understanding cost structures, supply chains, and what drives purchase decisions.

Engineering Challenge

Balancing product quality against unit economics while managing supplier relationships, production timelines, and brand positioning without enterprise resources.

Design Process

Defined target customer, developed brand identity, sourced manufacturing options, built cost models, tested marketing channels, and iterated based on customer feedback.

Technical Implementation

Managed end-to-end product lifecycle: supplier communication, manufacturing decisions, budgeting, packaging design, and direct customer outreach.

Engineering Decisions

Optimized for learnings-per-dollar in early batches rather than maximum margin. Prioritized supplier reliability over lowest unit cost to protect brand reputation.

Future Development

Scale production with validated demand, expand product line based on customer data, and apply manufacturing lessons to future hardware ventures.
